Візуалізація **Forced Layout Graph** — це динамічне та привабливе представлення взаємопов’язаних даних. Він використовує примусово спрямований алгоритм компонування для візуальної передачі зв’язків між об’єктами в наборі даних.

## Структура даних графіка примусового макету

Щоб представити графік примусової компоновки, ви можете використовувати такий інтерфейс `NodeLinkData`:

```typescript
{{Node}}

{{Link}}

{{NodeLinkData}}

```
### Ключові атрибути

- **name:** Унікальний ідентифікатор для вузла.
- **category:** класифікує вузол у певну групу або кластер.
- **джерело:** Ідентифікує вихідний вузол посилання.
- **target:** Ідентифікує цільовий вузол посилання.
- **value:** представляє числове значення, пов’язане з посиланням, яке вказує на те, наскільки сильно зв’язані вузли.

### Плюси і мінуси

#### плюси
- **Візуалізація зв’язків:** чітко представляє зв’язки між об’єктами через зв’язані вузли та зв’язки.

- **Динамічний макет:** використовує спрямований макет для органічного та візуально привабливого представлення.

#### мінуси
- **Складність із щільними графіками.** У щільно взаємопов’язаних графіках візуальний безлад може вплинути на читабельність.

- **Обмежено мережевими даними:** Найефективніше для візуалізації мережевих або зв’язаних даних; може не підходити для всіх типів наборів даних.
