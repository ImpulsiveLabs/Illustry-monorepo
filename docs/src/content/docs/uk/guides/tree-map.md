Візуалізація **Tree Map** — це динамічний та ієрархічний спосіб представлення даних, який забезпечує чіткий огляд складних структур. Використовуючи вкладені прямокутники, Treemap ефективно візуалізує ієрархічні зв’язки даних, що робить його безцінним інструментом для передачі як цілого, так і частин у наборі даних.

## Деревоподібна структура даних карти

Щоб представити дані карти дерева, ви можете використовувати такий інтерфейс `HierarchyData`:

```typescript
{{HierarchyNode}}

{{HierarchyData}}
```
### Ключові атрибути

- **ім’я:** ім’я вузла.
- **value:** число, яке представляє метрику для визначення розміру цього вузла.
- **category:** Описує категорію або тип, до якого належить вузол.
- **children:** Ця властивість є необов’язковою та представляє масив об’єктів HierarchyNode. Це означає, що вузол може мати дочірні вузли, створюючи ієрархічну структуру. Кожен дочірній вузол має однакову структуру HierarchyNode, що дозволяє представляти деревоподібну ієрархію.

### Плюси і мінуси

#### плюси
- **Розуміння ієрархії:** Treemaps чудово відображають ієрархічні структури даних, надаючи візуальну ієрархію, яка допомагає зрозуміти зв’язки між різними рівнями.

- **Пропорційне представлення:** Пропорційний розмір прямокутників дозволяє інтуїтивно зрозуміло відображати відносні значення або розміри різних категорій даних.

- **Ефективність простору:** Treemaps ефективно використовують простір, дозволяючи користувачам візуалізувати великі набори даних, не захаращуючи екран.

- **Швидка ідентифікація шаблонів:** користувачі можуть швидко ідентифікувати шаблони, викиди та концентрації в наборі даних за допомогою візуального розташування прямокутників.

#### мінуси
- **Складність із мітками, що накладаються:** У випадках, коли мітки накладаються, особливо в густонаселених Treemaps, читабельність може бути проблемою.

- **Складність із точністю:** Хоча Treemaps надають широкий огляд, досягнення точних порівнянь між окремими точками даних може бути складним через ієрархічний характер.

- **Обмежено ієрархічними даними: ** Деревоподібні карти найбільш ефективні під час візуалізації ієрархічних даних; для неієрархічних наборів даних альтернативні візуалізації можуть бути більш придатними.
