การแสดงภาพ **แผนผังต้นไม้** เป็นวิธีการแสดงข้อมูลแบบไดนามิกและเป็นลำดับชั้น ซึ่งให้ภาพรวมที่ชัดเจนของโครงสร้างที่ซับซ้อน ด้วยการใช้สี่เหลี่ยมที่ซ้อนกัน Treemap จะแสดงภาพความสัมพันธ์ของข้อมูลแบบลำดับชั้นได้อย่างมีประสิทธิภาพ ทำให้เป็นเครื่องมืออันล้ำค่าสำหรับการถ่ายทอดทั้งข้อมูลทั้งหมดและบางส่วนภายในชุดข้อมูล

## โครงสร้างข้อมูลแผนที่ต้นไม้

หากต้องการแสดงข้อมูลแผนที่ต้นไม้ คุณสามารถใช้อินเทอร์เฟซ `HierarchyData` ต่อไปนี้:

```typescript
{{HierarchyNode}}

{{HierarchyData}}
```
### คุณสมบัติที่สำคัญ

- **ชื่อ:** ชื่อโหนด
- **ค่า:** ตัวเลขที่แสดงถึงเมตริกเพื่อกำหนดขนาดของโหนดนั้น
- **หมวดหมู่:** อธิบายหมวดหมู่หรือประเภทที่โหนดนั้นอยู่
- **children:** คุณสมบัตินี้เป็นทางเลือกและแสดงถึงอาร์เรย์ของออบเจ็กต์ HierarchyNode มันบ่งบอกว่าโหนดสามารถมีโหนดลูกได้ โดยสร้างโครงสร้างแบบลำดับชั้น แต่ละโหนดย่อยมีโครงสร้าง HierarchyNode เดียวกัน ซึ่งช่วยให้สามารถแสดงลำดับชั้นที่เหมือนต้นไม้ได้

### ข้อดีข้อเสีย

#### ข้อดี
- **ความเข้าใจเชิงลำดับชั้น:** ทรีแมปเก่งในการแสดงโครงสร้างข้อมูลแบบลำดับชั้น โดยให้ลำดับชั้นที่มองเห็นได้ซึ่งช่วยในการทำความเข้าใจความสัมพันธ์ระหว่างระดับต่างๆ

- **การแสดงตามสัดส่วน:** การกำหนดขนาดตามสัดส่วนของสี่เหลี่ยมช่วยให้แสดงค่าสัมพัทธ์หรือขนาดของหมวดหมู่ข้อมูลต่างๆ ได้อย่างง่ายดาย

- **ประสิทธิภาพพื้นที่:** ทรีแมปใช้พื้นที่อย่างมีประสิทธิภาพ ช่วยให้ผู้ใช้เห็นภาพชุดข้อมูลขนาดใหญ่ได้โดยไม่เกะกะหน้าจอ

- **การระบุรูปแบบอย่างรวดเร็ว:** ผู้ใช้สามารถระบุรูปแบบ ค่าผิดปกติ และความเข้มข้นภายในชุดข้อมูลได้อย่างรวดเร็ว ผ่านการจัดเรียงสี่เหลี่ยมด้วยภาพ

#### ข้อเสีย
- **ความซับซ้อนของป้ายกำกับที่ทับซ้อนกัน:** ในกรณีที่ป้ายกำกับซ้อนทับกัน โดยเฉพาะอย่างยิ่งใน Treemap ที่มีประชากรหนาแน่น ความสามารถในการอ่านอาจเป็นเรื่องท้าทาย

- **ความยากในความแม่นยำ:** แม้ว่า Treemap จะให้ภาพรวมที่กว้าง แต่การเปรียบเทียบที่แม่นยำระหว่างจุดข้อมูลแต่ละจุดอาจเป็นเรื่องที่ท้าทายเนื่องจากมีลักษณะเป็นลำดับชั้น

- **จำกัดเฉพาะข้อมูลแบบลำดับชั้น:** ทรีแมปจะมีประสิทธิภาพมากที่สุดเมื่อแสดงข้อมูลแบบลำดับชั้น สำหรับชุดข้อมูลที่ไม่มีลำดับชั้น การแสดงภาพทางเลือกอาจมีความเหมาะสมมากกว่า
