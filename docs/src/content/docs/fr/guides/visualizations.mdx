# Visualisations dans votre espace de travail

Lorsque vous travaillez avec des visualisations, le système prend en charge une structure de données flexible via l'interface `VisualizationData` :

## Structure des données de visualisation

Pour représenter une visualisation dans votre système, vous pouvez utiliser l'interface `VisualizationData` suivante :

```typescript

{{VisualizationData}}

```

### Attributs clés
- **projectName** : le projet actif.
- **type :** Une chaîne ou un tableau de chaînes qui fournit le type de visualisation (le tableau ne fonctionne que pour les JSON).
- **name :** Un identifiant unique pour la visualisation.
- **description :** Un champ facultatif pour fournir un contexte ou des détails supplémentaires sur la visualisation.
- **createdAt :** L'horodatage indiquant la date à laquelle la visualisation a été initialement créée.
- **updatedAt :** L'horodatage indiquant la dernière modification de la visualisation.
- **isActive :** Un indicateur booléen spécifiant si le projet est actuellement actif.
- **tags :** Un champ facultatif utilisé pour catégoriser les visualisations.
- **data :** Les données que la visualisation présentera
